# 인간 경영지원 업무 완전 자동화 프로젝트 전략서

## 0. 한 줄 정의
인간 경영지원 조직의 반복·판단·조정·보고 업무를 **에이전트 운영체계(AOS: Autonomous Operations System)**로 재설계해, 사람은 최종 승인·예외처리·관계/정치적 판단에만 남기고 나머지는 자동 실행하는 프로젝트.

---

## 1. 목표

### 최종 목표
- 재무, 인사, 법무지원, 총무, 구매, 일정조율, 문서작성, 보고체계, 규정운영, 계약관리, 채용운영, 교육운영, 평가운영, 민원/문의 응대 등 **경영지원의 전 과정을 자동화**한다.
- 단순 자동화가 아니라, **업무 단위를 자동 분해하고 우선순위화하며 승인흐름까지 제어하는 디지털 운영조직**을 만든다.
- 사람은 아래 3가지에만 집중한다.
  1. 정책 결정
  2. 고위험 승인
  3. 관계/분쟁/예외 판단

### 정량 목표
- 경영지원 반복 업무 자동화율 85%+
- 문서 초안 자동 생성율 95%+
- 승인 전처리 자동화율 90%+
- 평균 처리시간 70% 단축
- 운영 누락/지연 80% 감소
- 감사추적 100% 로그화

---

## 2. 핵심 전제

### 전제 1. 자동화 대상은 "직무"가 아니라 "업무 원자단위"
사람의 직무를 그대로 대체하려고 하면 실패한다. 아래처럼 쪼개야 한다.
- 요청 접수
- 분류
- 정책 조회
- 리스크 스코어링
- 필요 문서 수집
- 초안 작성
- 승인권자 식별
- 승인 요청
- 집행
- 결과 보고
- 로그 기록
- 예외 escalation

### 전제 2. 100% 자율보다 100% 통제 가능한 자율이 우선
- 모든 자동 실행은 정책·권한·감사 로그 하에 움직여야 한다.
- 완전자율은 최종 상태가 아니라, **완전 추적 가능한 자율**이 먼저다.

### 전제 3. 경영지원 자동화의 본질은 AI 챗봇이 아니라 운영체계
핵심은 예쁜 대화형 UI가 아니라 아래 5가지다.
- 정책 엔진
- 워크플로 엔진
- 권한/승인 엔진
- 기록/감사 엔진
- 통합 실행 레이어

## 4. 시스템 비전: AOS (Autonomous Operations System)

## 4.1 구성 원칙
1. **한 개의 두뇌가 아니라 역할별 에이전트 조직**
2. **모든 행동은 정책과 권한으로 제한**
3. **모든 결정은 로그와 근거를 남김**
4. **고위험 액션은 항상 인간 승인 게이트 통과**
5. **실패를 전제로 롤백/재시도/에스컬레이션 설계**

## 4.2 에이전트 구조

### A. Intake Agent
- 이메일, 메신저, 폼, 문서함, ERP 이벤트를 수집
- 요청을 업무 유형별로 분류
- 긴급도/중요도/위험도를 태깅

### B. Policy Agent
- 사규, 운영규정, 계약정책, 예산원칙, 승인권한표 조회
- 특정 요청이 어떤 정책을 따라야 하는지 판정

### C. Workflow Agent
- 업무를 세부 단계로 분해
- 필요한 입력자료와 승인 흐름 설계
- 담당 에이전트 또는 인간에게 작업 배분

### D. Drafting Agent
- 품의서, 계약 초안, 메일, 보고서, 회의자료 초안 작성

### E. Verification Agent
- 누락/충돌/정책 위반 가능성 검토
- 금액, 날짜, 상대방, 법적 조건, 계정권한 등 교차검증

### F. Approval Agent
- 승인권자 식별
- 승인 요청 생성
- SLA 초과 시 리마인드
- 승인/반려 사유 기록

### G. Execution Agent
- ERP, HRIS, 그룹웨어, 캘린더, 드라이브, 전자결재, 회계도구 등 실제 시스템 액션 수행

### H. Audit Agent
- 전 과정 로그 기록
- 변경 전후 비교
- 사후 감사용 타임라인 생성

### I. Exception Agent
- 정책 애매함, 데이터 불일치, 고위험 건, 분쟁 가능성 감지 시 인간에게 에스컬레이션

## 5. 기술 아키텍처

## 5.1 계층 구조
1. **채널 계층**
   - 이메일
   - Discord/Slack/Telegram
   - 웹 포털
   - Google Workspace/Microsoft 365
   - ERP/회계/HR 시스템

2. **오케스트레이션 계층**
   - 이벤트 버스
   - 작업 큐
   - 상태 머신
   - 스케줄러
   - 재시도/백오프

3. **에이전트 계층**
   - 역할별 LLM 에이전트
   - 룰 기반 검증기
   - 도구 실행기

4. **정책/권한 계층**
   - 승인 매트릭스
   - 역할 기반 권한
   - 리스크 스코어 기준
   - 금액/법무/인사 민감도 규칙

5. **데이터/기억 계층**
   - 문서 저장소
   - 업무 상태 DB
   - 벡터 검색
   - 감사 로그 스토어
   - 결정 히스토리

6. **감사/관제 계층**
   - 관리자 콘솔
   - 케이스 타임라인
   - 실패 큐
   - 경고/알림
   - 품질 리포트

## 5.2 필수 기능
- 모든 업무건에 고유 Case ID 부여
- 모든 에이전트 행동에 근거(source) 저장
- 승인 없는 고위험 액션 차단
- 변경 diff 기록
- 인간 인수인계 원클릭
- 감사 리플레이 가능

## 7. 도입 전략

## Phase 1. 문서/보고 자동화
가장 빨리 성과가 나는 영역부터.
- 회의록
- 품의 초안
- 공지/안내문 초안
- 계약/제안서 1차 초안
- 보고서 자동 편집

**성과:** 생산성 즉시 상승, 신뢰 확보 쉬움

## Phase 2. 승인 전처리 자동화
- 요청 분류
- 증빙 체크
- 승인권자 추천
- 결재 패키지 자동 생성

**성과:** 병목 해소

## Phase 3. 시스템 실행 자동화
- 캘린더 생성
- 계정 생성 요청
- 구매 발주 생성
- 결제 일정 등록
- 온보딩 태스크 자동 발행

**성과:** 운영인력 직접 절감 시작

## Phase 4. 정책 기반 자율 운영
- 예외 없는 반복 업무는 자동 승인/자동 집행
- 사람은 예외 큐만 처리

**성과:** 경영지원 조직의 구조 자체가 바뀜

## Phase 5. 경영지원 자율조직
- 에이전트들이 도메인별로 협업
- 스스로 병목 감지
- KPI 악화 시 운영 개선안 제안
- 조직운영/비용/처리속도 최적화

## 13. 대표 보고용 결론
이 프로젝트는 "경영지원 챗봇"을 만드는 일이 아니다.
이건 **인간 경영지원 조직을 에이전트 기반 운영체계로 재구성하는 일**이다.

승부처는 세 가지다.
1. 업무를 잘게 쪼개는 설계력
2. 정책/권한/감사로 자율을 통제하는 통제력
3. 문서 생성이 아니라 실제 실행까지 연결하는 통합력

가장 좋은 전략은 아래다.
- 1단계: 문서/보고 자동화로 즉시 성과 확보
- 2단계: 승인 전처리 자동화로 병목 제거
- 3단계: 실제 시스템 실행 자동화로 비용구조 전환
- 4단계: 예외 중심 운영조직으로 경영지원 인력 구조 자체 재설계

이 프로젝트가 성공하면,
경영지원은 더 이상 "사람이 처리하는 지원부서"가 아니라,
**정책과 에이전트가 운영하는 자율 운영 시스템**이 된다.
